Introdução

  • Razões para replicação:
    • Confiabilidade -> Manter várias cópias possibilita melhor proteção contra dados corrompidos.
    • Desempenho -> O desempenho de um SD precisa ser garantido quando ele é ampliado
      • Ampliação em quantidade - maior número de processos acessando um mesmo recurso. A replicação garante o balanceamento de carga entre os recursos.
      • Ampliação geográfica - processos dispersos tendem a ter maior tempo de resposta. A replicação pode mover cópia do recurso para perto dos consumidores.

Problema da replicação:

  • Inconsistência:
    • Como manter as réplicas idênticas?
    • Se uma cópia é modificada, a mudança deve ser feita em todas as outras cópias.
    • Manter cópias atualizadas requer, portanto, maior capacidade de comunicação (largura de banda).

Modelos de Consistência Centrado em Dados

  • Depósito de dados é um termo usado para generalizar quaisquer mecanismos de armazenamento, como memória compartilhada, bancos de dados, ou sistema de arquivos.
  • Um depósito de dados pode ser distribuído fisicamente por várias máquinas.
  • A premissa é que cada processo que pode acessar dados do depósito tem uma cópia local (ou próxima) disponível do depósito inteiro.
  • Operações de escrita no depósito são propagadas para todas as cópias.
  • Um modelo de consistência é, em essência, um contrato entre os processos e o depósito.
  • Normalmente, um processo que executa uma operação de leitura sobre um item de dados espera que a operação retorne o valor escrito pela última operação de escrita sobre aquele dado.
  • Na ausência de um relógio global, é difícil definir com precisão qual operação de escrita é a última.

Consistência contínua

  • A replicação de dados propõe problemas de consistência que não podem ser resolvidos com eficiência de modo geral.
  • Somente abrandando a consistência, ou seja, definindo regras para a tolerância de inconsistências, é que podemos ter certa eficiência.
  • A Consistência contínua se refere a 3 eixos de inconsistências:
    • Desvio de valores numéricos entre réplicas
    • Desvio de idade entre réplicas
    • Desvio em relação à ordenação de operações de atualização.
  • A medida de inconsistência em termos de desvios numéricos pode ser utilizada por aplicações para as quais os dados têm semântica numérica.
    • Exemplo: duas cópias não devem se desviar por mais de 0,02.

  • Os desvios de idade estão relacionados com a última vez que uma réplica foi atualizada.
    • Algumas aplicações podem tolerar que uma réplica forneça dados antigos, contanto que não sejam demasiadamente antigos.
